## Используемый датасет

В проекте используется файл `dataset.csv`, расположенный в одной директории с Python-скриптами. Набор данных содержит **150 строк и 9 столбцов**.

Техническая колонка:

- `Unnamed: 0` — индекс исходной таблицы, не используется в аналитических расчётах.

Числовые колонки:

- `platelets`;
- `serum creatinine`;
- `serum sodium`;
- `creatinine phosphokinase`;
- `ejection fraction`.

Категориальные колонки:

- `sex`;
- `smoking`;
- `death`.

Колонки `sex`, `smoking` и `death` представлены числами, но рассматриваются как категориальные признаки, потому что принимают ограниченный набор значений, а операция сложения для них не имеет содержательного смысла.

## Запуск отдельных модулей

### 1. Первичный анализ данных

```bash
python dataset.py
```

При запуске модуль:

- загружает `dataset.csv`;
- выводит форму датасета;
- показывает типы данных;
- считает количество пропусков;
- рассчитывает среднее, медиану и стандартное отклонение для числовых колонок;
- выводит частоты значений для категориальных колонок;
- сохраняет тот же отчёт в файл `report.txt`.

При импорте `dataset.py` отчёт не печатается, но глобальная переменная `df` доступна другим модулям.

### 2. Первичная визуализация

```bash
python data_scatter.py
```

Модуль открывает окно с точечной диаграммой. В этом режиме используются только числовые колонки. Пользователь может выбирать колонку для оси X и колонку для оси Y, после чего график автоматически перестраивается.

Кнопка **«Сохранить»** сохраняет текущую диаграмму в PNG-файл с именем вида:

```text
graphHH_MM_SS.png
```

### 3. Улучшенная визуализация

```bash
python data_visual.py
```

Модуль поддерживает числовые и категориальные колонки, а тип графика выбирается автоматически.

| Выбранные данные | Тип графика |
|---|---|
| две разные числовые колонки | точечная диаграмма |
| одна и та же числовая колонка | гистограмма |
| одна и та же категориальная колонка | круговая диаграмма |
| категориальная колонка на X, числовая на Y | столбчатая диаграмма |
| числовая колонка на X, категориальная на Y | коробочная диаграмма |

В верхней части окна находится выпадающий список цветовых схем `matplotlib`. По умолчанию для фамилии на букву `С` выбрана схема `GnBu`.

### 4. Рисование поверх графиков

```bash
python data_draw.py
```

Это основная версия приложения. Она расширяет `data_visual.py` и добавляет режим ручной модификации графиков.

Возможности режима рисования:

- включение и выключение кнопкой **«Рисовать»**;
- выбор цвета через стандартное окно `tkinter`;
- настройка толщины кисти через `Spinbox`;
- рисование при удержании левой кнопки мыши;
- выход из режима рисования при повторном нажатии кнопки, правом клике или смене колонок;
- отмена последней завершённой линии по `Ctrl+Z`;
- сохранение PNG вместе с нанесёнными пометками.

## Smoke-тест основного приложения

В `data_draw.py` предусмотрен автоматический smoke-тест, который создаёт график, добавляет тестовую линию и сохраняет результат:

```bash
python data_draw.py --smoke-test --smoke-output test_outputs/manual_smoke.png
```

В среде без графического дисплея на Linux можно использовать `xvfb-run`:

```bash
xvfb-run -a python3 data_draw.py --smoke-test --smoke-output test_outputs/manual_smoke.png
```

## Автоматические тесты

Запуск тестов:

```bash
python -m pytest -q
```

На Linux без активного графического окружения:

```bash
xvfb-run -a python3 -m pytest -q
```

Тесты проверяют:

- расчёт толщины кисти, RGB-цвета и HEX-представления цвета;
- создание приложения `DataDrawApp`;
- сохранение изображения в smoke-тесте;
- работу отмены последней линии;
- корректное переключение на круговую диаграмму для категориального признака.

## Краткое описание архитектуры

Проект разделён на несколько логических уровней.

`config.py` хранит индивидуальные параметры студента: первую букву фамилии, студенческий ID и таблицу соответствия рекурсивной суммы цифр маркерам `matplotlib`.

`dataset.py` отвечает за слой данных: загружает CSV-файл, классифицирует колонки, формирует статистический отчёт и предоставляет глобальную переменную `df`.

`data_scatter.py` реализует базовый графический интерфейс для точечной диаграммы. График строится с помощью `matplotlib`, преобразуется в изображение и выводится на `tkinter.Canvas`.

`data_visual.py` добавляет профилирование колонок, автоматический выбор типа графика и работу с цветовыми схемами. Для построения изображения используется `FigureCanvasAgg`, после чего результат передаётся в интерфейс через `Pillow`.

`data_draw.py` наследует расширенное приложение визуализации и добавляет слой ручной разметки. Нарисованные линии хранятся как объекты `Stroke`, а при сохранении переносятся на итоговое PNG-изображение.
